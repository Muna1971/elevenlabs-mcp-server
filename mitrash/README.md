# مطراش — مرحبا الساع

> تطبيق مراسلات إماراتي محلي بهوية أصيلة

**مطراش** تطبيق مراسلات ويب فوري مستوحى من التراث الإماراتي (حيث كان "المطراش" قديماً هو حامل الرسائل الذي ينقل الأخبار بين الناس). يهدف المشروع إلى بناء منصة مراسلات محلية تعكس الهوية الثقافية الإماراتية مع معايير تقنية عالمية.

## الحالة الحالية: MVP (نموذج أولي)

هذه المرحلة الأولى من المشروع — خادم بسيط لإثبات الجدوى التقنية. الميزات المتقدمة ستُضاف تدريجياً.

### ما يعمل حالياً

- تسجيل دخول بسيط باسم المستخدم (بدون كلمة مرور في هذه المرحلة)
- محادثات فورية 1-على-1 عبر WebSocket
- قائمة المستخدمين المتصلين في الوقت الحقيقي
- حفظ سجل المحادثات بشكل دائم (JSON)
- إعادة الاتصال التلقائية عند انقطاع الشبكة
- واجهة عربية كاملة (RTL) بتصميم إماراتي فاخر
- تصنيف الرسائل حسب التاريخ
- تنبيه للرسائل الجديدة
- تصميم متجاوب (يعمل على الجوال والحاسوب)

### ما لم يُنجز بعد (خارطة الطريق)

- [ ] **التسجيل والمصادقة الحقيقية** (OTP عبر الجوال + UAE PASS)
- [ ] **التشفير من طرف إلى طرف** (Signal Protocol)
- [ ] **إرسال الوسائط** (صور، فيديو، صوت، ملفات)
- [ ] **المجموعات والقنوات**
- [ ] **المكالمات الصوتية والمرئية** (WebRTC)
- [ ] **الإشعارات الفورية** (Web Push)
- [ ] **المحفظة الرقمية** (العيدية، دفع الفواتير)
- [ ] **المساعد الذكي باللهجة الإماراتية**
- [ ] **الملصقات التعبيرية المحلية**
- [ ] **تطبيقات الجوال** (Flutter/React Native)
- [ ] **ترقية قاعدة البيانات** (من JSON إلى PostgreSQL)
- [ ] **الربط مع الخدمات الحكومية**

## البنية التقنية

```
mitrash/
├── server/              # الخادم (Node.js)
│   ├── index.js         # Express + WebSocket
│   └── db.js            # طبقة التخزين (JSON)
├── public/              # الواجهة الأمامية
│   ├── index.html       # صفحة الترحيب والدخول
│   ├── chat.html        # واجهة المحادثة
│   ├── css/
│   │   └── style.css    # التصميم
│   └── js/
│       ├── welcome.js   # منطق صفحة الترحيب
│       └── chat.js      # منطق المحادثة
├── docs/                # التوثيق
├── package.json
└── README.md
```

### المكدس التقني

| الطبقة | التقنية |
|---|---|
| الخادم | Node.js + Express |
| المراسلة الفورية | WebSocket (ws) |
| التخزين | JSON file (مؤقتاً) |
| الواجهة | HTML5 + CSS3 + Vanilla JS |
| الخطوط | Cairo + Amiri (Google Fonts) |

## التشغيل

### المتطلبات

- Node.js 18 أو أحدث
- npm

### الخطوات

```bash
# 1. انتقل إلى مجلد المشروع
cd mitrash

# 2. ثبّت المكتبات
npm install

# 3. شغّل الخادم
npm start
```

ثم افتح المتصفح على: `http://localhost:3000`

### وضع التطوير (مع إعادة التحميل التلقائي)

```bash
npm run dev
```

### اختبار المحادثة بين مستخدمين

1. افتح المتصفح على `http://localhost:3000` وسجّل بـ "أحمد"
2. افتح نافذة خاصة (Incognito) على نفس الرابط وسجّل بـ "محمد"
3. ستظهر كل جلسة للأخرى في قائمة "المتصلون الآن"
4. اضغط على اسم المستخدم الآخر وابدأ المحادثة

## الأمان والخصوصية

> تحذير: هذا الإصدار **نموذج أولي للعرض فقط**، وليس جاهزاً للإنتاج.

القيود الحالية:

- لا توجد مصادقة حقيقية (أي شخص يمكنه استخدام أي اسم)
- لا يوجد تشفير من طرف إلى طرف
- البيانات مخزنة في ملف JSON بدون تشفير
- لا يوجد حد لمعدل الطلبات (Rate Limiting)

قبل النشر للإنتاج يجب:

1. إضافة مصادقة قوية (OTP + UAE PASS)
2. تطبيق Signal Protocol للتشفير
3. ترقية قاعدة البيانات إلى PostgreSQL مشفّرة
4. تفعيل HTTPS/WSS
5. إضافة Rate Limiting و DDoS protection
6. مراجعة أمنية شاملة

## الخيارات المعمارية

### لماذا Node.js + WebSocket؟

- **سرعة التطوير:** JavaScript على طرفي الواجهة والخادم
- **أداء ممتاز:** WebSocket يوفر مراسلة فورية منخفضة التأخير
- **مجتمع ضخم:** توفر مكتبات جاهزة
- **سهولة النشر:** على أي منصة سحابية

### لماذا JSON File للتخزين؟

- **بساطة الإعداد:** صفر تبعيات أصلية (native dependencies)
- **يعمل فوراً:** لا حاجة لتثبيت قاعدة بيانات
- **مناسب للـ MVP فقط:** سيُستبدل بـ PostgreSQL عند الانتقال للإنتاج

### لماذا Vanilla JS بدلاً من React/Vue؟

- **حجم أصغر:** لا يحتاج bundler
- **سرعة تحميل فائقة:** مناسب لواجهة بسيطة
- **سهولة الفهم:** أي مطور يمكنه قراءة الكود
- **الترقية لاحقاً:** يمكن الانتقال إلى Svelte/React عند الحاجة
